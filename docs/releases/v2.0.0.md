# OnScreen v2.0.0

The "all-in-one media server" release — closing the headline gaps vs Plex/Emby/Jellyfin while keeping OnScreen's homelab-first, no-cloud-dependency stance. Live TV/DVR is now a first-class media type alongside movies/TV/music/photos. Self-hosted SSO replaces the old hosted-OAuth providers. Hardware encoding fans out to NVENC, QSV, VAAPI, and AV1. The plugin system lands as outbound MCP — OnScreen calls out to your tools, not the other way around.

113 commits since `v1.1.2`. Three security passes closed. Test posture is the strongest in the project's life: 87 UAT, 73 integration, fuzz tests for parsers, strong unit coverage across api, settings, observability, valkey, streaming, middleware, and the auth surface. Live beta has been up under load since 2026-03-31.

---

## Highlights

- **SSO without the cloud.** OIDC + SAML 2.0 + LDAP/AD configurable from the admin UI. Auto-generated SP keypair on first SAML save. Replaces v1's hosted-OAuth (Google/GitHub/Discord) which fit a SaaS shape, not self-hosting.
- **Live TV + DVR.** HDHomeRun + M3U tuners, XMLTV and Schedules Direct EPG, channel guide grid, scheduled recordings with series rules + conflict resolution, retention-purge cron. Recordings land in a normal library and play end-to-end.
- **MCP plugin system.** Outbound only — OnScreen calls *out* to plugins (notifications, metadata, tasks). Per-role allow-list, host pinning, IP-pin to prevent SSRF. CRUD admin UI under `/settings/plugins`.
- **Hardware encoding fanout.** NVENC, QSV, VAAPI, AMF for H.264/HEVC; AV1 hardware encode on RTX 40+ and Intel ARC; OpenCL HDR-to-SDR tonemap; PGS/VOBSUB → WebVTT OCR pipeline.
- **Audiophile-grade music.** MusicBrainz metadata, Cover Art Archive fallback, ReplayGain, embedded + sidecar `.lrc` lyrics with synced display, hi-res FLAC direct stream, gapless transitions via two-element rotation.
- **Photos as a first-class library.** EXIF-derived ordering, on-demand JPEG/HEIC resize via `/items/{id}/image`, photo-album playlists.
- **People/cast pages, intro/credits markers, trickplay scrubbing, per-user library access, parental controls, request system** — the rest of the long tail.

## What's new by area

### Auth — self-hosted SSO

Replaces v1's Google/GitHub/Discord OAuth (which made sense for SaaS but not self-hosting) with three protocols every homelab IdP speaks.

- **OIDC** — generic, tested against Authentik and Keycloak. Issuer URL + client ID/secret in the SSO settings page; user-provisioning on first sign-in.
- **SAML 2.0** — SP-initiated, signed AuthnRequests with RSA-SHA256, in-memory request tracker keyed by RelayState (so cookie-SameSite gotchas don't break the round-trip behind reverse proxies). SP keypair auto-generated on first enable.
- **LDAP / Active Directory** — bind-as-user with service account, `(uid={username})`-style filter, optional StartTLS / LDAPS, admin group DN for auto-promotion.
- **Forgot-password** with SMTP delivery and session-epoch revocation (every password change kicks out every existing session, every device).
- **Password policy** — 12-char floor across all set/reset paths.
- **Device pairing** flow for TV/console clients (no on-screen keyboard required).

### Live TV / DVR

Closes Track D from the v2 roadmap.

- HDHomeRun and M3U tuner abstractions, HLS proxy.
- Channel guide grid, EPG via XMLTV (free) or Schedules Direct ($35/yr).
- DVR scheduler with series rules, "new episodes only," conflict resolution when tuner count is exceeded.
- Recording retention purge as a scheduled task.
- Channel callsign auto-match on lineup import.

### Plugins — outbound MCP

OnScreen calls *out* to plugin servers (Anthropic's Model Context Protocol). Inbound MCP was rejected for security — plugins can't drive OnScreen, only respond when called.

- Roles: `notification`, `metadata`, `task`. Each plugin declares its role at registration.
- Per-role allow-list keeps notification plugins out of metadata fan-out paths and vice versa.
- `allowed_hosts` JSONB column + IP-pin in the egress client prevent SSRF and DNS-rebinding attacks.
- Admin CRUD + role/enabled filter at `/settings/plugins`.

### Transcoding

- **AV1 hardware encode** on `av1_nvenc` (RTX 40+) and `av1_qsv` (Intel ARC). Detect-and-offer; never required.
- **HEVC hardware encode** across NVENC, QSV, VAAPI, AMF.
- **HDR-to-SDR tonemap** with OpenCL acceleration; gracefully falls back to software path on hosts without OpenCL/zscale.
- **Subtitle burn-in** for image-based subs (PGS/VOBSUB) when the client can't render WebVTT.
- **PGS/VOBSUB → WebVTT OCR** via tesseract — exposed as both a per-stream admin action *and* a scheduler task for library back-fills.
- **Trickplay** seekbar thumbnails generated as a background scheduler task.

### Music

- MusicBrainz as the primary metadata source.
- **Cover Art Archive fallback** when TheAudioDB has no art for a release.
- **Embedded + sidecar `.lrc` lyrics** with synced timestamp display in the track-info modal.
- **Gapless playback** via dual-`<audio>` element rotation — the next track's bytes are buffered and codec-init done before the current one ends.
- ReplayGain track + album modes.
- Multi-disc albums collapse into one logical album with disc/track ordering preserved.
- Compilation albums attribute "Various Artists" at the album level while individual tracks keep their original artist.
- Artist artwork discovery for flat layouts and per-folder layouts.

### Photos

- New first-class library type (not stuffed under movies).
- EXIF-driven ordering and metadata.
- On-demand JPEG/HEIC resize via `/items/{id}/image?w=...` — no pre-rendered thumbnails to manage.
- Photo albums as ordered playlists.

### Frontend

- **Continue Watching** + **Recently Added** per library on the home hub.
- **Mobile player** — sticky transport controls above the bottom nav, viewport-fit playback, touch-reachable scrubber.
- **Theme toggle** (light/dark/system) with `data-theme` attribute and a no-flash bootstrap script.
- **Notifications** with bell icon, unread badge, SSE stream.
- **Search** — type-filter chips, in-library indicator on Discover results, request flow integrated.
- **Playlists**, **collections**, **favorites**, **history**, **per-user library access**, **parental rating ceilings on managed profiles**.
- **People/cast pages** with TMDB credits.
- **Intro/credits markers** with admin detection controls and player skip buttons.

### Operator surface

- **Built-in HTTPS** via operator-provided PEM (`TLS_CERT_FILE` + `TLS_KEY_FILE`); half-set is rejected at startup.
- **OTel tracing** with sample ratio + endpoint configurable from the admin UI.
- **Schedules Direct + XMLTV EPG**, **Cover Art Archive**, **OpenSubtitles** all routed through `safehttp` (host allow-list, IP-pin, no redirects).
- **Admin backup / restore** with dry-run round-trip integration test.
- **Scheduled tasks** admin page (`/settings/tasks`) — cron-driven admin jobs (rescan, OCR back-fill, DVR retention, EPG refresh).
- **Worker fleet** — embedded transcode worker plus optional remote workers, with per-worker capability advertising.
- **Discovery** UDP beacon for LAN clients.

### Imports / interop

- **NFO sidecar import** (Kodi-style) — `movie.nfo` / `tvshow.nfo` / `episodedetails.nfo` / `album.nfo` / `artist.nfo`. NFO is authoritative when present; falls back to TMDB/MusicBrainz when missing.

## Security posture

Three independent deep-scan passes closed before tag, plus the bug-fix passes during the live beta soak. Highlights:

- Trickplay endpoint is now auth-gated (was anonymous in v1.x).
- Password reset invalidates every active session and refresh token via session-epoch bump.
- Demote / role-change revokes the target's tokens within ~1 segment.
- Segment-token revocation on credential rotation (HLS playlists can't be replayed by old credentials).
- Bearer / API-key never accepted as query param — header-only, no log leakage.
- LDAP filter escaping; OIDC state/nonce/PKCE; SAML cookie name + refresh fixed.
- CORS Allow-Credentials never paired with `Origin: *`.
- Recover middleware doesn't leak panic value.
- Path traversal probes return 403/404 across `/segments/`, `/artwork/`, `/trickplay/`, `/media/stream/`.
- All outbound URL fetches (artwork, OpenSubtitles, plugins, webhooks) routed through `safehttp` with host allow-list + IP-pin.

Manual + automated re-probe layer at [docs/manual-test-plan.md](https://github.com/CollinJAycock/OnScreen/blob/main/docs/manual-test-plan.md) and [web/tests/e2e/security.spec.ts](https://github.com/CollinJAycock/OnScreen/blob/main/web/tests/e2e/security.spec.ts).

## Deferred to v2.1

Documented in [docs/v2-roadmap.md](https://github.com/CollinJAycock/OnScreen/blob/main/docs/v2-roadmap.md), not gaps:

- **Books / comics** — CBR/CBZ/EPUB parsing + reader UI; smallest user overlap so it took the cut.
- **Tidal / Qobuz full integration** — scoped down to "metadata supplement" for v2; full streaming integration in v2.1.
- **Hardware bit-perfect playback** — lands with the native client phase (browser audio pipelines can't deliver this).
- **Native client apps** (Windows/macOS/Linux/iOS/Android/TV).
- **Async OCR endpoint** — the synchronous `POST /items/{id}/subtitles/ocr` is request-bound and 524s behind reverse proxies with sub-multi-minute response timeouts (Cloudflare Tunnel free tier = 100s). Workaround for v2.0: trigger the scheduler `ocr_subtitles` task instead. v2.1 converts the per-stream endpoint to job-queued pattern.
- **Podcast RSS auto-fetch** — out of scope per project memo (OnScreen doesn't download content).

## Upgrade notes

Coming from v1.1.x:

- **Database migrations are forward-compatible.** `goose up` against your existing DB will land all new schema (auth tokens table changes, plugin registry, DVR, photo library, intro markers, lyrics, audit). Backup before upgrading regardless — `make migrate-status` shows where you are.
- **Hosted-OAuth users** (anyone who had Google/GitHub/Discord OAuth configured): those handlers were removed and replaced with generic OIDC + LDAP. Migrate via the SSO settings page — point an Authentik / Keycloak / Auth0 / Azure AD instance at OnScreen's OIDC callback. There is no automatic migration; users must sign in with the new provider on first visit (account auto-created if email matches).
- **TMDB API key** is still required for movie metadata enrichment.
- **Build images** rebuilt: `docker build -f docker/Dockerfile.ffmpeg -t onscreen-ffmpeg:latest .` before `docker build -f docker/Dockerfile.gpu`. The FFmpeg base now bundles tesseract for OCR.
- **Reverse-proxy operators** — built-in HTTPS available (`TLS_CERT_FILE` + `TLS_KEY_FILE`); HSTS only emitted when the request is actually over HTTPS, per RFC 6797 §7.2.

## Known limitations

- **Local SAML testing over HTTP** requires the in-memory request tracker that ships in v2.0 (cookie-backed tracker breaks on cross-port `localhost:*` POSTs). HTTPS deployments are unaffected.
- **PGS/VOBSUB OCR per-stream** behind Cloudflare Tunnel free tier hits the 100s response timeout. Use the scheduler `ocr_subtitles` task for a library back-fill instead. Fix queued for v2.1.
- **DSD audio files** are detected but not playable (browser audio pipelines don't support DSD); they fail gracefully rather than silently.
